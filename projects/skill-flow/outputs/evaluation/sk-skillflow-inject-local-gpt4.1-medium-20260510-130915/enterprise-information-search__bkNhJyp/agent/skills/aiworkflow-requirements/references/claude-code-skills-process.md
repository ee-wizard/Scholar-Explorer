# Skill 作成・更新プロセス

> 本ドキュメントは統合システム設計仕様書の一部です。
> 管理: .claude/skills/aiworkflow-requirements/
>
> **重要**: 本仕様は Anthropic公式 skill-creator を最重要資料として策定し、プロジェクト固有の拡張を追加する。

---

## スキル作成・更新プロセス

### プロセス概要

**新規作成フロー**:

```
1. 具体例でスキルを理解する
       ↓
2. 再利用可能なスキルコンテンツを計画する
       ↓
3. スキルを初期化する（init_skill.mjs）
       ↓
4. スキルを編集する（リソース実装 + SKILL.md 作成）
       ↓
5. スキルを検証する（quick_validate.mjs）
       ↓
6. 実際の使用に基づいてイテレーション
```

**更新フロー**:

```
1. 更新トリガーを特定する
       ↓
2. 影響範囲を分析する
       ↓
3. 更新を実装する
       ↓
4. 検証する（quick_validate.mjs）
       ↓
5. フィードバックを記録する（log_usage.mjs）
```

### Step 1: 具体例でスキルを理解する

スキルの使用パターンが既に明確に理解されている場合のみスキップ。既存スキルでも価値がある。

効果的なスキルを作成するには、スキルがどのように使用されるかの具体例を明確に理解する。

**確認すべき質問**:

- 「このスキルはどのような機能をサポートすべきか？」
- 「このスキルがどのように使用されるか、いくつか例を挙げられるか？」
- 「このスキルをトリガーするためにユーザーは何と言うか？」
- 「参照すべき書籍や知識体系は何か？」

**このステップの完了条件**: スキルがサポートすべき機能の明確な理解が得られた時点。

### Step 2: 再利用可能なスキルコンテンツを計画する（追記）

具体例を効果的なスキルに変換するため、各例を分析：

1. 例をゼロから実行する方法を検討
2. これらのワークフローを繰り返し実行する際に役立つスクリプト、参照資料、素材を特定
3. 参照すべき書籍を特定（コンテキスト圧縮）
4. Task分割（agents/）を必須分析対象にする

**分析パターン**:

| パターン             | 問題                         | 解決策                                       |
| -------------------- | ---------------------------- | -------------------------------------------- |
| コード重複           | 同じコードを毎回書き直す     | `scripts/{{script-name}}.mjs` にスクリプト化 |
| ボイラープレート重複 | 同じテンプレートを毎回作成   | `assets/{{template-name}}/` に素材化         |
| 知識再発見           | 同じ情報を毎回調査           | `references/{{reference-name}}.md` に参照化  |
| 原則適用             | 同じ判断基準を毎回思い出す   | `references` に書籍参照として知識体系を圧縮  |
| 思考ログ肥大         | 探索・試行錯誤がメインに残る | agents/ に Task化して隔離                    |
| フェーズ混線         | リサーチと生成が同窓で混ざる | Taskをフェーズ単位で分ける                   |
| 知識ベタ書き         | SKILL/agents が肥大          | references へ外部化                          |

### Step 3: スキルを初期化する

開発中のスキルが既に存在し、イテレーションが必要な場合のみスキップ。

新規スキルを作成する場合、常に `init_skill.mjs` を実行：

```bash
Node scripts/init_skill.mjs <skill-name> --path .claude/skills
```

スクリプトの実行内容：

- 指定パスにスキルディレクトリを作成
- 適切な frontmatter と TODO プレースホルダを含む SKILL.md テンプレートを生成
- 例示リソースディレクトリを作成：`scripts/`, `references/`, `assets/`

初期化後、生成された SKILL.md と例示ファイルを必要に応じてカスタマイズまたは削除。

### Step 4: スキルを編集する

スキルは別の Claude インスタンスが使用するために作成される。Claude に有益で非自明な情報を含める。

#### 再利用可能コンテンツから開始

実装は Step 2 で特定した再利用可能リソースから開始：`scripts/`, `references/`, `assets/`

**スクリプトのテスト**: 追加したスクリプトは実際に実行してバグがないことと期待通りの出力を確認。類似スクリプトが多い場合は代表的なサンプルのみテスト。

**不要なディレクトリの削除**: 初期化スクリプトが作成した例示ファイルで不要なものは削除。

#### SKILL.md を更新（改訂：references フィールド削除・description統合）

**Frontmatter テンプレ（改訂版）**

```yaml
---
name: {{skill-name}}
description: |
  {{スキルの機能説明（2〜3行）}}

  • {{ドキュメント/ルール/スキーマ/書籍名など}} / 適用: {{適用範囲}} / 目的: {{目的}}
  • {{同上}} / 適用: {{適用範囲}} / 目的: {{目的}}

  Trigger:
    Use when {{発動条件}}.
    {{word1}}, ..., {{wordN}}
allowed-tools:
  - {{tool1}}
  ...
  - {{toolN}}
---
```

**本文の作成（追記：Taskナビ必須）**

- agents/ の各Taskを「いつ」「何の入力で」「何を返すか」導線として書く
- references/ は「読む条件」を明記（常時読み込み禁止）
- scripts/ は「引数」「期待出力」「失敗時の扱い」を明記

### Step 5: スキルを検証する

開発完了後、検証スクリプトを実行：

```bash
Node scripts/quick_validate.mjs .claude/skills/<skill-name>
```

**検証項目**:

- SKILL.md の存在
- YAML frontmatter の形式
- 必須フィールド（name, description）の存在
- name のハイフンケース規則
- description の文字数制限と禁止文字

### Step 6: イテレーション

スキルをテストした後、ユーザーが改善を要求することがある。

**イテレーションワークフロー**:

1. 実際のタスクでスキルを使用
2. 苦戦や非効率に気づく
3. SKILL.md またはバンドルリソースをどう更新すべきか特定
4. 変更を実装して再テスト
5. `scripts/log_usage.mjs` でフィードバックを記録

### スキル更新プロセス

既存スキルの更新・改善を行うプロセス。新規作成とは異なり、既存の構造を維持しながら改善を加える。

#### 更新トリガー

| トリガー           | 説明                                        | 対応                           |
| ------------------ | ------------------------------------------- | ------------------------------ |
| フィードバック蓄積 | LOGS.md に改善点が蓄積                      | パターンを分析し優先度を決定   |
| 仕様変更           | 参照書籍の新版、API変更、ドメイン知識の更新 | 影響範囲を特定し段階的に更新   |
| パフォーマンス問題 | スキルの実行効率が低下                      | ボトルネックを特定し最適化     |
| 使用パターン変化   | 想定外の使用方法が主流に                    | description と発動条件を見直し |
| 依存スキル更新     | 依存先スキルの変更による影響                | 互換性を確認し必要に応じて更新 |

#### 影響範囲分析

更新前に以下を確認：

| 確認項目             | 確認方法                                     |
| -------------------- | -------------------------------------------- |
| 依存エージェント     | skill_list.md でスキルを使用するエージェント |
| 依存スキル           | dependencies フィールドの参照先              |
| スクリプト利用状況   | scripts/ 内のファイルを呼び出すワークフロー  |
| references/ 参照状況 | SKILL.md からのリンク                        |

#### 更新タイプ別ワークフロー

**Type A: コンテンツ更新（低リスク）**

対象: 誤字修正、説明文の改善、参照書籍の追加

```
1. SKILL.md を直接編集
2. quick_validate.mjs で検証
3. log_usage.mjs で記録
```

**Type B: 構造更新（中リスク）**

対象: セクション追加、ワークフロー変更、スクリプト追加

```
1. 変更計画を立案（影響範囲を明記）
2. バックアップを作成（git stash または別ブランチ）
3. 段階的に変更を実装
4. 各段階で quick_validate.mjs を実行
5. 依存エージェントでの動作確認
6. log_usage.mjs で記録
```

**Type C: 破壊的更新（高リスク）**

対象: name 変更、description の大幅変更、ディレクトリ構造変更

```
1. 影響を受けるすべてのエージェント・コマンドをリストアップ
2. マイグレーション計画を作成
3. 新旧両方のパスで一時的に動作するよう移行期間を設ける
4. 依存先を順次更新
5. 旧パスを削除
6. skill_list.md を更新
7. log_usage.mjs で記録
```

#### 更新時の注意事項

| 注意事項                       | 理由                                     |
| ------------------------------ | ---------------------------------------- |
| description は慎重に変更       | 発動条件が変わりエージェントの挙動に影響 |
| name 変更は最終手段            | すべての参照先の更新が必要               |
| scripts/ の互換性を維持        | 引数・出力形式の変更は依存先に影響       |
| references/ の削除前に確認     | SKILL.md からのリンク切れを防ぐ          |
| 更新後は必ずフィードバック記録 | 継続的改善のためのデータ蓄積             |

## フィードバックループ

### 目的

スキルが使用されるたびにフィードバックを収集し、継続的に改善する仕組みを提供する。

### 実装方式

**採用方式**: エージェント責務方式（明示的な記録フェーズ）

**理由**:

- 確実性が最も高い（ワークフローに組み込むため記録漏れがない）
- AIの不確実性を排除（スクリプトで確実に実行）
- メンテナンス性が高い（エージェントテンプレートで標準化）

### 標準ログスクリプト: log_usage.mjs

**ファイル**: `scripts/log_usage.mjs`

**責務**:

| 処理           | 説明                                                 |
| -------------- | ---------------------------------------------------- |
| 実行記録の追記 | 使用履歴をログファイルに記録                         |
| メトリクス更新 | 使用回数・成功率を更新                               |
| レベル評価     | 現在のメトリクスがレベルアップ条件を満たすかチェック |

**引数**:

| 引数       | 必須 | 説明                       |
| ---------- | ---- | -------------------------- |
| `--result` | 必須 | `success` または `failure` |
| `--phase`  | 任意 | 実行したフェーズ名         |
| `--agent`  | 任意 | 実行したエージェント名     |
| `--notes`  | 任意 | 追加のフィードバックメモ   |

**使用形式**:

```bash
node .claude/skills/{{skill-name}}/scripts/log_usage.mjs \
  --result {{success|failure}} \
  --phase "{{phase-name}}" \
  --agent "{{agent-name}}"
```

## 品質基準

### スキル作成時のチェックリスト

| チェック項目                                 | 検証方法                               |
| -------------------------------------------- | -------------------------------------- |
| SKILL.md が 500 行以内                       | `wc -l SKILL.md`                       |
| YAML frontmatter が有効                      | quick_validate.py                      |
| name がハイフンケース（最大64文字）          | 正規表現で検証                         |
| description が 1024 文字以内                 | 文字数カウント                         |
| 書籍参照が含まれている                       | references フィールド確認              |
| 不要な補助ドキュメントがない                 | README.md 等の不在確認                 |
| references/ のファイルが SKILL.md からリンク | grep で検証                            |
| agents/\*.md が「仕様書」になっている        | 役割/入力/出力/制約/参照が揃っているか |
| agents に知識本文が埋め込まれていない        | 大段落がある場合 references へ移動     |
| description に Anchors と Trigger がある     | 1024文字以内、Markdown不使用           |
| references/ は SKILL.md から直リンク         | grep で検証                            |

### ベストプラクティス要約

| カテゴリ               | ベストプラクティス                                      |
| ---------------------- | ------------------------------------------------------- |
| 簡潔さ                 | 説明より例を優先、Claude が既に知っていることは書かない |
| 書籍参照               | 1-3冊の著名書籍で知識体系を圧縮参照                     |
| 自由度                 | タスクの脆弱性に応じて具体性を調整                      |
| Progressive Disclosure | SKILL.md を軽量に、詳細は references/ に                |
| 重複回避               | 情報は SKILL.md か references/ のどちらか一方のみ       |
| スクリプト             | 繰り返し書くコードはスクリプト化、必ずテスト            |
| 発動条件               | description に明確に記述（本文ではなく）                |
| 相対パス               | references/ へのリンクは相対パスで記述                  |

## 命名規則

### スキル名

| 規則                       | パターン                                  |
| -------------------------- | ----------------------------------------- |
| ハイフンケース             | `{{domain}}-{{topic}}`                    |
| 具体的で明確               | 曖昧な単語（optimization, utils）を避ける |
| 動詞+名詞 または 名詞+名詞 | `{{verb}}-{{noun}}` / `{{noun}}-{{noun}}` |
| 最大64文字                 | 簡潔に                                    |

### ファイル名

| ファイル種別 | 命名規則                                                  |
| ------------ | --------------------------------------------------------- |
| スクリプト   | `{{verb}}_{{target}}.py` または `{{verb}}_{{target}}.mjs` |
| 参照資料     | `{{topic}}.md` または `{{domain}}.md`                     |
| アセット     | `{{purpose}}.{{ext}}` または `{{asset-name}}/`            |

## ファイル参照形式

### 相対パスの記述規則

**SKILL.md からの相対パス形式を使用**:

| 参照対象   | パス形式                 |
| ---------- | ------------------------ |
| スクリプト | `scripts/{{file}}.py`    |
| 参照資料   | `references/{{file}}.md` |
| アセット   | `assets/{{file}}`        |

**SKILL.md での参照記述形式**:

```markdown
## Resources

- **{{リソース1名}}**: See [references/{{resource-1}}.md](references/{{resource-1}}.md)
- **{{リソース2名}}**: See [references/{{resource-2}}.md](references/{{resource-2}}.md)
- **{{リソースN名}}**: See [references/{{resource-n}}.md](references/{{resource-n}}.md)

## Scripts

- `scripts/{{script-1}}.py`: {{スクリプト1の説明}}
- `scripts/{{script-n}}.mjs`: {{スクリプトNの説明}}
```

### 禁止される参照形式

| 形式                 | 問題点                                 |
| -------------------- | -------------------------------------- |
| スキル名のみ         | パスが不明確で読み込みできない         |
| 絶対パス             | 環境依存（ユーザーごとにパスが異なる） |
| `../` を含む相対パス | 曖昧さが生じ、正確なパス解決ができない |

## skill_list.md 仕様

### 目的

利用可能な全スキルのインデックスを提供し、スキルの発見性とエージェントへの割り当てを効率化する。

### ファイル配置

`.claude/skills/skill_list.md`

### 記述形式

```markdown
## {{番号}}. {{エージェント役割名}}

- **エージェント名:** `@{{agent-name}}`
- **エージェントの配置:** `.claude/agents/{{agent-name}}.md`

| スキル名           | パス                                     | 概要                |
| ------------------ | ---------------------------------------- | ------------------- |
| **{{skill-name}}** | `.claude/skills/{{skill-name}}/SKILL.md` | {{20-40文字の概要}} |
```

### 更新ルール

| ルール             | 説明                                                 |
| ------------------ | ---------------------------------------------------- |
| スキル作成時       | 対応するエージェントセクションにエントリを追加       |
| エージェント作成時 | 新規見出しセクションを追加                           |
| 共有スキル追加時   | 共通スキルセクションに追加し、使用エージェントを明記 |
| 陳腐化スキル削除時 | エントリを削除                                       |
