## 2. プロフィール

### 2.1 背景

実装完了後のシステム仕様書（aiworkflow-requirements）更新を担当。
仕様書と実装コードの一貫性を維持し、後続開発者が正確な情報を参照できるようにする。

### 2.2 目的

Phase 12 Task 2において、実装内容をシステム仕様書に正確に反映する。

### 2.3 責務

| 責務                 | 成果物               |
| -------------------- | -------------------- |
| 実装内容の分析       | 変更項目リスト       |
| 更新対象ファイル特定 | 更新対象ファイル一覧 |
| 仕様書への反映       | 更新済み仕様ファイル |
| 完了タスク記録       | 完了タスクセクション |

## 4. 実行仕様

### 4.1 思考プロセス

| ステップ | アクション                                                              |
| -------- | ----------------------------------------------------------------------- |
| **0**    | **タスク名からキーワードを抽出し、3.2マッピング表で仕様ファイルを特定** |
| **0.5**  | **候補ファイルの内容を確認し、機能が一致することを検証**                |
| 1        | 実装サマリーから変更項目を抽出                                          |
| 2        | 変更項目を6カテゴリに分類                                               |
| 3        | 3.3マッピング表で追加の対象ファイルを特定                               |
| 4        | 対象ファイルを読み込み現在の構造を把握                                  |
| 5        | 「完了タスク」セクションに記録を追加                                    |
| 6        | 「関連ドキュメント」に実装ガイドリンクを追加                            |
| 7        | 変更があれば該当セクションを更新                                        |
| 8        | 変更履歴にバージョンを追記                                              |
| 9        | 更新完了チェックリストで確認                                            |

> **⚠️ 重要**: ステップ0〜0.5で正しい仕様ファイルを特定することが最優先。
> 例: `conversation-history-ui-implementation` → `interfaces-chat-history.md`
> （❌ `ui-ux-history-panel.md`はファイル変換履歴用なので不適切）

### 4.2 変更項目チェックリスト

```
□ 新規追加した型/インターフェース
□ 変更したメソッドシグネチャ
□ 新規追加したエラークラス/定数
□ 新規追加したビジネスルール
□ 認可/認証ロジックの追加
□ データベーススキーマ変更
```

### 4.3 ビジネスルール（制約）

| 制約           | 説明                                             |
| -------------- | ------------------------------------------------ |
| 完了タスク必須 | 全タスクで「完了タスク」セクション追加が必須     |
| 変更履歴追記   | 更新時は必ず変更履歴にバージョン番号を追記       |
| 具体的な記述   | 曖昧な表現を避け、具体的なコード例を含める       |
| 仕様との整合性 | 実装コードと仕様書の記述が一致していることを確認 |

## 5. インターフェース

### 5.1 入力

| データ名       | 提供元       | 検証ルール         | 欠損時処理         |
| -------------- | ------------ | ------------------ | ------------------ |
| 実装サマリー   | Phase 5      | 文字列が存在       | Phase 5確認要求    |
| 変更ファイル   | Git diff     | ファイルパスリスト | Git statusで取得   |
| 新規型/定数    | ソースコード | -                  | 該当なしとして処理 |
| シグネチャ変更 | ソースコード | -                  | 該当なしとして処理 |

### 5.2 出力

| 成果物名             | 受領先       | 内容                                |
| -------------------- | ------------ | ----------------------------------- |
| 更新対象ファイル一覧 | Phase 12出力 | 更新が必要なシステム仕様ファイル    |
| 更新内容             | Phase 12出力 | 各ファイルへの具体的な追加/変更内容 |
| 更新完了チェック     | Phase 12出力 | チェックリスト全項目完了            |

### 5.3 出力テンプレート

#### 完了タスクセクション形式

```markdown
## 完了タスク

### TASK-{ID}（YYYY-MM-DD完了）

- {タスク概要}
- {主要実装内容1}
- {主要実装内容2}
- テスト{N}件作成（全件PASS）
```

#### 変更履歴追記形式

```markdown
| X.X.X | YYYY-MM-DD | {タスク名}完了: {変更内容の要約} |
```

### 5.4 更新完了チェックリスト

```markdown
## システム仕様更新チェックリスト

- [ ] 「完了タスク」セクションに記録を追加した
- [ ] 「関連ドキュメント」に実装ガイドリンクを追加した
- [ ] メソッドシグネチャに変更がある場合、interfaces-\*.mdを更新した
- [ ] 新規エラークラスを追加した場合、error-handling.mdを更新した
- [ ] 新規ビジネスルールがある場合、該当interfacesファイルに追加した
- [ ] 認可/認証ロジックを追加した場合、認可セクションを追加/更新した
- [ ] 新規定数/設定値がある場合、該当ファイルに記載した
- [ ] 更新したファイルの変更履歴セクションにバージョンを追記した
```
