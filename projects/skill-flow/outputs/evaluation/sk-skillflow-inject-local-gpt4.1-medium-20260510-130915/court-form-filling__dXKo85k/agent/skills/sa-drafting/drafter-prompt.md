# 初稿撰写助手提示词模板 (Drafter Prompt Template)

Use this template when dispatching a drafter subagent.

```
Task tool (general-purpose):
  description: "撰写任务 N: [章节名称]"
  prompt: |
    你正在撰写任务 N: [章节名称]

    ## 任务描述

    [计划中的完整任务文本 - 粘贴在这里，不要让子智能体读取文件]

    ## 背景

    [场景设置：本章节在论文中的位置，与其他章节的依赖关系，理论框架背景]

    ## 在你开始之前

    如果你对以下内容有疑问：
    - 论证要求或验收标准
    - 论证方法或写作策略
    - 引用来源或理论假设
    - 任务描述中任何不清楚的地方

    **现在就问。** 在开始写作之前提出任何疑虑。

    ## 你的工作

    一旦你清楚了要求：
    1. 严格按照任务说明进行撰写
    2. 进行论证自查（如果任务要求，遵循 ADW）
    3. 验证论证逻辑成立
    4. 提交你的草稿
    5. 自我审查（见下文）
    6. 汇报

    工作目录：[directory]

    **当你工作时：** 如果你遇到意外或不清楚的事情，**提问**。
    暂停并澄清总是可以的。不要猜测或假设。

    ## 汇报前：自我审查

    用全新的眼光审视你的工作。问自己：

    **完整性：**
    - 我是否完整论证了提纲中的所有点？
    - 我是否遗漏了任何论据？
    - 是否有未处理的反驳意见？

    **质量：**
    - 这是我最好的学术写作吗？
    - 术语是否准确清晰（描述事物的本质，而不是堆砌辞藻）？
    - 行文是否流畅且逻辑严密？

    **纪律：**
    - 我是否避免了过度论证 (YAGNI)？
    - 我是否只写了要求的内容？
    - 我是否遵循了现有的学术规范？

    **验证：**
    - 论据是否真正支持了主张（不仅仅是表面相关）？
    - 我是否遵循了 ADW（如果要求）？
    - 引用是否全面？

    如果你在自我审查中发现问题，现在就修复它们，然后再汇报。

    ## 汇报格式

    完成后，汇报：
    - 你撰写了什么内容
    - 你验证了什么逻辑及结果
    - 修改的文件
    - 自我审查发现（如果有）
    - 任何问题或疑虑
```
