# Tailoring Examples

Use these examples to understand how to rewrite resume bullets to match a Job Description (JD) without fabricating facts.

## Example 1: Software Engineer -> SRE Role
**Goal:** Emphasize reliability and infrastructure over feature shipping.

**Job Requirement:** "Experience with CI/CD pipelines and improving system reliability (99.9% uptime)."
**Source Bullet (Master Resume):** "Deployed new features to production weekly using Jenkins and fixed bugs in the payments service."

**✅ Good Rewrite (Tailored):**
"Managed weekly production deployments via Jenkins CI/CD pipelines, contributing to payment service stability and reliability."
*(Reasoning: Shifts focus from "features" to "pipelines" and "stability" to align with SRE keywords, using only true facts.)*

**❌ Bad Rewrite (Fabricated):**
"Architected a CI/CD pipeline that achieved 99.9% uptime for the payments service."
*(Reasoning: FABRICATION. The source never mentioned "architecting" or the specific "99.9%" metric. The JD asked for it, but we cannot invent it.)*

## Example 2: Marketing -> Product Manager (Career Transition)
**Goal:** Translate domain-specific tasks into transferable product skills.

**Job Requirement:** "Cross-functional leadership and stakeholder management."
**Source Bullet (Master Resume):** "Coordinated with sales and design teams to launch the Q3 ad campaign."

**✅ Good Rewrite (Tailored):**
"Led cross-functional collaboration between sales and design stakeholders to deliver the Q3 ad campaign launch."
*(Reasoning: Replaces "coordinated" with "led cross-functional collaboration" and "teams" with "stakeholders" to match PM terminology.)*

**❌ Bad Rewrite (Vague/Weak):**
"Talked to people to get the ads done."
*(Reasoning: Too informal, loses professional impact.)*
