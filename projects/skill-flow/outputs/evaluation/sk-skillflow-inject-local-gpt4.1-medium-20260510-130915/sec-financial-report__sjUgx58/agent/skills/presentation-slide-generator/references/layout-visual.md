# レイアウト・ビジュアルガイドライン

**責務**: レイアウト法則・分割パターン・図形・矢印・余白・統一感

---

## 1. 読み順の原則

### 基本ルール

スライドは**左上から右下**の順に読まれる。

```
┌─────────────────────────────────────┐
│  1 ───────────→ 2                   │
│  │                │                 │
│  ↓                ↓                 │
│  3 ───────────→ 4                   │
└─────────────────────────────────────┘
```

### 重要な情報の配置

| 位置 | 優先度 | 配置する内容 |
|------|--------|-------------|
| 左上 | 最高 | 最重要情報・タイトル |
| 右上 | 高 | 結論・数字 |
| 左下 | 中 | 補足情報・根拠 |
| 右下 | 低 | 注釈・出典 |

## 2. 分割パターン

### 2分割（比較・対比）

```
┌─────────────┬─────────────┐
│             │             │
│    左側     │    右側     │
│   (Before)  │   (After)   │
│             │             │
└─────────────┴─────────────┘
```

**用途**: 比較、Before/After、対立する概念

### 4分割（複数要素）

```
┌─────────────┬─────────────┐
│      1      │      2      │
├─────────────┼─────────────┤
│      3      │      4      │
└─────────────┴─────────────┘
```

**用途**: 4つの要素、2x2マトリックス、4象限

### 上下分割

```
┌─────────────────────────────┐
│           上部              │
│      （主要コンテンツ）     │
├─────────────────────────────┤
│           下部              │
│       （補足・注釈）        │
└─────────────────────────────┘
```

**用途**: メイン情報と補足の分離

## 3. 矢印の使い分け

### 矢印タイプ

| 矢印 | 名称 | 用途 | 形状例 |
|------|------|------|--------|
| 直線矢印 | ストレート | 因果関係・単純な流れ | → |
| 鍵線矢印 | エルボー | 要素間の接続 | └→ ┌→ |
| 三角矢印 | ブロック | 全体の流れ・強調 | ▶ ▷ |
| 双方向矢印 | ダブル | 相互関係 | ↔ |
| 曲線矢印 | カーブ | 循環・回転 | ↻ |

### 使い分けガイド

```
【因果関係】
原因 ────→ 結果

【プロセス】
STEP1 ──▶ STEP2 ──▶ STEP3

【要素間の接続】
┌── 要素A
│
└→ 要素B

【全体の流れ（三角矢印）】
背景 ▷ 課題 ▷ 解決策 ▷ 効果
```

## 4. 図形の使い分け

### 図形と表現対象

| 図形 | 表現対象 | 例 |
|------|---------|-----|
| 四角形 | 具体的な要素 | 組織、システム、製品、ステップ |
| 楕円 | 抽象的な概念 | ビジョン、価値観、目標、開始/終了 |
| 角丸四角 | 中間的な要素 | プロセス、ステップ、ボタン |
| 三角形 | 階層構造 | ピラミッド、優先度、方向性 |
| ひし形 | 判断・分岐 | 条件分岐、Yes/No |
| 六角形 | 連携・結合 | ハブ、中心要素 |

### 使い分けルール

```
【具体 → 四角形】
┌─────────────┐
│  営業部門   │
└─────────────┘

【抽象 → 楕円】
    ╭─────────╮
    │ ビジョン │
    ╰─────────╯

【プロセス → 角丸四角】
╭─────────────╮
│  STEP 1    │
╰─────────────╯

【判断 → ひし形】
    ◇
   /  \
  Yes  No
```

## 5. 余白ルール

### 図形の余白最小化

```
【NG】余白が大きすぎる
┌───────────────────────────┐
│                           │
│      テキスト             │
│                           │
└───────────────────────────┘

【OK】余白を最小化
┌───────────────┐
│  テキスト     │
└───────────────┘
```

### 余白の目安

| 要素 | 内側余白 | 外側余白 |
|------|---------|---------|
| 図形内テキスト | 8-16px | - |
| 図形間 | - | 16-24px |
| スライド端 | - | 32-48px |

## 6. 統一感の作り方

### 色の統一

| 用途 | カラー | 統一ルール |
|------|--------|-----------|
| メインカラー | `--wave-blue` | 重要な要素・タイトル |
| アクセント1 | `--sakura-pink` | 注意喚起・課題 |
| アクセント2 | `--wave-aqua` | 解決策・ポジティブ |
| アクセント3 | `--autumn-yellow` | 強調・数字 |
| 背景 | `--bg-dim` | カード・ボックス |

### 順番の統一

スライド間で同じカテゴリの要素は同じ順番で表示：

```
【スライドA】
1. コスト削減
2. 生産性向上
3. 品質改善

【スライドB】（同じ順番を維持）
1. コスト削減の詳細
2. 生産性向上の詳細
3. 品質改善の詳細
```

### ページ番号

すべてのスライドにページ番号を記入（表紙を除く）：

```
┌─────────────────────────────┐
│                             │
│        コンテンツ           │
│                             │
├─────────────────────────────┤
│                        2/10 │
└─────────────────────────────┘
```

## 7. パンくずリスト

### 全体の流れを示す

スライド上部に現在位置を表示：

```
【現在位置を強調】
背景 > 課題 > 【解決策】 > 効果 > 結論

【CSSでの実装例】
.breadcrumb .current {
  font-weight: bold;
  color: var(--wave-blue);
}
```

## 8. セクション目次

### セクションごとに挟む

```
┌─────────────────────────────────┐
│                                 │
│   1. 背景                       │
│   2. 課題  ← 現在               │
│   3. 解決策                     │
│   4. 効果                       │
│                                 │
└─────────────────────────────────┘
```

### 目次での強調方法

| 方法 | 実装 |
|------|------|
| 色で強調 | 現在セクションを`--wave-blue`に |
| 矢印で示す | → 課題 |
| 番号を大きく | 現在の番号を拡大 |

## 10. カードレイアウト最適化

### タイトル改行防止

カード内のタイトル（h4）は**一行で表示**されるべき。改行が入ると視認性が低下する。

```css
/* タイトルは改行しない */
.list-item h4,
.flow-step h4,
.grid-card h3 {
  white-space: nowrap;
}

/* 説明文は改行を許可 */
.list-item span,
.flow-step span {
  white-space: normal;
  line-height: 1.4;
}
```

### カード幅とフォントサイズのバランス

| 項目 | 推奨値 | 注意点 |
|------|--------|--------|
| カード最小幅 | 280px (list), 180px (flow) | タイトルが一行で収まる幅 |
| カード最大幅 | 380px (list), 220px (flow) | 横並び時のバランス |
| タイトルフォント | 1.2-1.4rem × font-scale | 大きすぎると改行発生 |
| 説明文フォント | 0.9-1.1rem × font-scale | 可読性を確保 |
| カード内gap | 0.5-0.8rem | 要素間の適切な間隔 |
| カードpadding | 1.2-1.8rem | 余白とのバランス |

### 画面表示用CSS例

```css
/* リストスライド - カード幅・フォントサイズ最適化 */
.slide-list .list-container {
  display: flex;
  gap: 1.5rem;
  justify-content: center;
}

.slide-list .list-item {
  flex: 1;
  min-width: 280px;
  max-width: 380px;
  padding: 1.8rem 1.5rem;
  gap: 0.8rem;
}

.slide-list .list-item h4 {
  font-size: calc(1.4rem * var(--font-scale));
  white-space: nowrap;  /* 改行防止 */
}

.slide-list .list-item span {
  font-size: calc(1.1rem * var(--font-scale));
  line-height: 1.5;
}
```

### 印刷用CSS例

```css
@media print {
  .slide-list .list-item h4 {
    font-size: 11pt !important;
    white-space: nowrap !important;
  }

  .slide-list .list-item span {
    font-size: 9pt !important;
    line-height: 1.3 !important;
    white-space: normal !important;
  }
}
```

- [ ] フォントサイズが大きすぎないか
