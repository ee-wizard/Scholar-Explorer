# 構造化データテンプレート（structure.md）

スライド生成時に出力する構造化データのテンプレート。
**index.htmlと完全同期**を維持し、改善・修正作業の基礎情報として使用する。

---

## 同期原則

```
【重要】index.html ⇔ structure.md は常に同期を維持すること

- HTMLを修正 → structure.mdの該当箇所も更新
- structure.mdを修正 → HTMLを再生成
- どちらか一方だけ変更して放置しない
```

## フォーマット説明

### 同期を確実にするポイント

1. **HTML構造セクション**: 各スライドの実際のHTMLを記録
   - index.html修正時は、この部分も同時に更新
   - structure.mdから再生成時は、この内容をそのまま使用

2. **スタイル調整セクション**: スライド固有のCSS調整を記録
   - テキスト切れ対策、フォントサイズ変更など

3. **調整履歴セクション**: 各スライドの変更履歴
   - いつ何を変更したかを記録

4. **同期チェックリスト**: 同期状態の確認用
   - 修正後に必ずチェック

### index.html修正時の手順

```
1. index.htmlを修正
2. structure.mdの該当スライドの「HTML構造」を更新
3. 必要に応じて「スタイル調整」を更新
4. 「調整履歴」に変更内容を追記
5. 「修正履歴」に記録（同期状態: ✅ 同期済）
6. 「同期チェックリスト」で確認
```

### structure.md修正時の手順

```
1. structure.mdの内容を修正
2. 「修正履歴」に記録（同期状態: ⚠️ 要再生成）
3. html-generator Taskでindex.htmlを再生成
4. 「修正履歴」を更新（同期状態: ✅ 同期済）
```
