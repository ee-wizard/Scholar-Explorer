# LLM/Script 責務分離ガイドライン

曖昧性を排除し、再現性を高めるため、処理を「LLM担当」と「Script担当」に明確に分離する。

---

## Script担当（決定論的・機械的処理）

**原則**: 入力が同じなら出力も同じ。曖昧性ゼロ。

| 処理 | スクリプト | 説明 |
|------|-----------|------|
| 構成案検証 | `validate-structure.mjs` | 23種のスライドタイプ・必須フィールド・アイコン形式を機械的に検証 |
| 視覚検証 | `verify-slides.mjs` | Playwrightでスクリーンショット撮影、レイアウト崩れを機械的に検出 |
| フィードバック記録 | `log_usage.mjs` | 使用状況を決まったフォーマットで記録 |
| HTMLテンプレート | `assets/slide-template.html` | 基本構造は固定テンプレートを使用 |
| 構造化データ形式 | `assets/structure-template.md` | structure.mdのフォーマットは固定 |

### Scriptが検証するもの

- スライドタイプが23種のいずれかに一致するか
- 必須フィールド（title, slides, type, message, icon）の存在
- アイコン形式が`fa-xxx`パターンに一致するか
- HTMLが正しく表示されるか（スクリーンショット撮影）

---

## LLM担当（創造的・判断が必要な処理）

**原則**: 文脈理解・創造性・柔軟な判断が必要な処理。

| 処理 | 担当Task | 説明 |
|------|---------|------|
| ヒアリング | `hearing-facilitator` | ユーザーとの対話、情報収集、曖昧な要求の明確化 |
| 構成設計 | `structure-designer` | 情報の分解、最適なスライドタイプの選択、アイコン選定 |
| HTML生成 | `html-generator` | 構成案を元にしたHTML組み立て、アニメーション適用 |
| 改善・修正 | `slide-modifier` | 修正要求の分析、影響範囲の判断、修正案設計 |

### LLMが判断するもの

- どの情報をどのスライドタイプで表現するか
- どこで改行するか（意味的な区切り）
- どのアイコンが内容に最も適切か
- アニメーションのタイミング調整
- ユーザー要求の意図解釈

## 責務境界の明確化

```
【検証の流れ】
LLM: structure.md作成 → Script: validate-structure.mjs検証 → LLM: エラー修正
LLM: index.html生成 → Script: verify-slides.mjs検証 → LLM: 問題修正

【決定論的にすべき処理をLLMに任せない】
❌ LLMがスライドタイプの妥当性を判断
✅ Scriptが23種のホワイトリストで機械的に検証

❌ LLMがアイコン形式の妥当性を判断
✅ Scriptが正規表現パターンで機械的に検証

❌ LLMが「なんとなく良さそう」で完了判断
✅ Scriptがスクリーンショット撮影で客観的に検証

【創造的にすべき処理をScriptに任せない】
❌ Scriptが「この内容はlistタイプ」と判定
✅ LLMが文脈を理解してタイプを選択

❌ Scriptが改行位置を自動決定
✅ LLMが意味的な区切りを判断して改行
```

                               │
                    エラーあり │ エラーなし
                               ▼
                        ┌──────────────┐

                        └──────────────┘

                               │
                    問題あり   │ 問題なし
                               ▼
                        ┌──────────────┐
                        │ LLM修正      │
                        │ or 完了      │
                        └──────────────┘

## 原則サマリー

| 判断基準 | Script担当 | LLM担当 |
|----------|-----------|---------|
| 入力→出力の確定性 | 確定的 | 非確定的 |
| ルールベース判断 | ✅ | ❌ |
| 文脈理解必要 | ❌ | ✅ |
| 創造性必要 | ❌ | ✅ |
| 再現性 | 100% | 可変 |
