# Section-by-Section Checklists

Combined quality checklists for paper writing and manuscript review.

---

## Pre-Writing Checklist

Before starting to write:

- [ ] Research question/contribution clearly defined
- [ ] Target venue identified (conference, journal)
- [ ] Formatting standard determined (IEEE, ACM)
- [ ] Page/word limit known
- [ ] Key related work identified
- [ ] Data/experiments completed
- [ ] Figures/tables drafted

---

## Abstract Checklist (150-250 words)

- [ ] Opens with broad context (1-2 sentences)
- [ ] States specific problem and gap
- [ ] Describes approach/methodology
- [ ] Reports key results quantitatively
- [ ] States significance/implications
- [ ] Self-contained (readable without full paper)
- [ ] No citations
- [ ] No undefined acronyms

---

## Introduction Checklist

### Structure
- [ ] Follows funnel: Broad -> Narrow -> Gap -> Hypothesis
- [ ] Broad context establishes importance (1-2 sentences)
- [ ] Narrows to specific problem area
- [ ] Gap statement explicit ("However, X remains unknown")
- [ ] Ends with clear hypothesis or objective

### Content
- [ ] Motivation is compelling (real-world relevance)
- [ ] Contributions listed (3-5 bullet points)
- [ ] Paper organization paragraph at end
- [ ] Not too broad or too narrow opening
- [ ] Literature review not overlong (details go in Related Work)

---

## Related Work Checklist

- [ ] Organized by theme or approach
- [ ] All relevant prior work cited
- [ ] Explicit comparisons ("Unlike X, our approach...")
- [ ] Gap positioning clear
- [ ] Your work's positioning stated
- [ ] No straw man arguments
- [ ] Recent and foundational work included

---

## Methodology/Approach Checklist

- [ ] Approach clearly described
- [ ] Design rationale explained (why this approach)
- [ ] Algorithms/procedures detailed
- [ ] Architecture diagrams included (if applicable)
- [ ] Reproducible from description
- [ ] Novel aspects highlighted
- [ ] All notation defined

---

## Results Checklist

### Structure
- [ ] Each experiment has: Question -> Approach -> Finding -> Interpretation
- [ ] Figures/tables in logical order
- [ ] Key findings highlighted

### Quantitative Rigor
- [ ] Statistics present (p-values, confidence intervals)
- [ ] Sample sizes stated (n values)
- [ ] Error bars/variance included
- [ ] Baselines compared
- [ ] Numbers, not vague terms ("3.2x improvement" not "significant improvement")

### Figures and Tables
- [ ] All figures/tables referenced in text
- [ ] Captions complete and informative
- [ ] Figures readable at print size
- [ ] Tables have proper formatting

---

## Discussion Checklist

### Structure
- [ ] Opens by revisiting hypothesis (supported/refuted)
- [ ] Interprets findings (doesn't just repeat Results)
- [ ] Places in literature context
- [ ] Acknowledges limitations
- [ ] Suggests future directions

### Content
- [ ] No overclaiming (claims match evidence)
- [ ] Speculation clearly labeled ("we speculate that...")
- [ ] Comparison to prior work explicit
- [ ] Limitations honest and substantive
- [ ] Future work concrete and actionable

---

## Conclusion Checklist

- [ ] Summarizes contributions (not results)
- [ ] States significance and impact
- [ ] Outlines future work
- [ ] Brief (1-2 paragraphs)
- [ ] No new information introduced

---

## Scientific Clarity Checklist

### Rigor
- [ ] Claims supported by data
- [ ] Statistics present for comparisons
- [ ] Sample sizes stated for all experiments
- [ ] Controls adequately described
- [ ] Variability measures included (SD/SEM)
- [ ] Replication stated (number of runs)

### Logic
- [ ] Hypothesis clearly stated
- [ ] Methods adequate for reproducibility
- [ ] Results ordered logically
- [ ] Interpretation proportional to evidence
- [ ] Limitations acknowledged
- [ ] Speculation labeled

### Language
- [ ] Terminology consistent throughout
- [ ] Abbreviations defined on first use
- [ ] Jargon minimized or explained
- [ ] Hedging appropriate to evidence
- [ ] Active voice where it aids clarity
- [ ] Quantitative language used

---

## Formatting Checklist (IEEE)

- [ ] Page size: A4 (210mm x 297mm)
- [ ] Margins: 19mm top, 43mm bottom, 14.32mm left/right
- [ ] Font: Times New Roman throughout
- [ ] Title: 24pt bold, centered
- [ ] Body: 10pt, justified, single-spaced
- [ ] Two-column layout with 4.22mm spacing
- [ ] Section headings: 10pt bold, numbered
- [ ] Figure captions below, "Fig. X." format
- [ ] Table captions above, "TABLE X" format
- [ ] References: numbered [1], [2], etc.
- [ ] All fonts embedded in PDF
- [ ] All acronyms defined on first use

---

## Formatting Checklist (ACM)

- [ ] Abstract: single column
- [ ] Body: two columns
- [ ] CCS Concepts section included
- [ ] Keywords section included (4-8 keywords)
- [ ] ACM Reference Format section included
- [ ] Font: Libertine or Times New Roman
- [ ] Section headings: Title Case, numbered
- [ ] Figure captions: "Figure X:" format
- [ ] Table captions: "Table X:" format
- [ ] References: full author names (not initials)
- [ ] DOIs included when available
- [ ] All fonts embedded in PDF

---

## Citation Checklist

- [ ] All claims supported by citations
- [ ] 15-20+ references for full paper
- [ ] Mix of recent (last 5 years) and foundational
- [ ] Consistent citation format
- [ ] DOI or URL included when available
- [ ] Self-citations appropriate (not excessive)
- [ ] No missing references
- [ ] All citations referenced in text

---

## Final Submission Checklist

### Content
- [ ] Abstract matches paper content
- [ ] Contributions clearly stated
- [ ] All figures/tables referenced
- [ ] No placeholder text remaining
- [ ] Acknowledgments complete
- [ ] Author information correct

### Quality
- [ ] Spell-check completed
- [ ] Grammar check completed
- [ ] Consistent terminology throughout
- [ ] No orphan headings
- [ ] Balanced column layout (final page)

### Technical
- [ ] PDF generated correctly
- [ ] All fonts embedded
- [ ] Links working (if applicable)
- [ ] File size acceptable
- [ ] Page limit met
- [ ] Blind review requirements met (if applicable)

---

## Review Completion Checklist

After reviewing a manuscript:

- [ ] Core message identified and verified
- [ ] Structure assessed against standard patterns
- [ ] Each section reviewed with specific checklist
- [ ] Scientific rigor audited
- [ ] Language polished
- [ ] Formatting verified
- [ ] Changes tracked or documented
- [ ] Summary of improvements provided
