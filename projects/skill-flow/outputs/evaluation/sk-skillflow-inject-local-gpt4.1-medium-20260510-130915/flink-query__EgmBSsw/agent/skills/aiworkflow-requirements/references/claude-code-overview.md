> 本ドキュメントは統合システム設計仕様書の一部です。

## 概要

### 本仕様書の目的

本仕様書は、Claude Code の3層アーキテクチャ（Command / Agent / Skill）の設計仕様を定義する。本仕様書に従うことで、100人中100人が同一の構造・品質のコンポーネントを作成できることを目標とする。

### 適用範囲

| 対象         | パス                           | 責務                     | 詳細仕様                           |
| ------------ | ------------------------------ | ------------------------ | ---------------------------------- |
| コマンド     | `.claude/commands/ai/*.md`     | ユーザーインターフェース | [18-commands.md](./18-commands.md) |
| エージェント | `.claude/agents/*.md`          | ワークフロー実行         | [18-agents.md](./18-agents.md)     |
| スキル       | `.claude/skills/[skill-name]/` | 知識・スクリプト提供     | [18-skills.md](./18-skills.md)     |

### 設計思想

| 原則               | 説明                                                     |
| ------------------ | -------------------------------------------------------- |
| 責務の完全分離     | 各層は単一責務を持ち、他層の責務を侵犯しない             |
| 依存の一方向性     | Command → Agent → Skill の方向のみ依存を許可する         |
| 確実な実行の分離   | AIの不確実性を排除すべき処理はスクリプトに委譲する       |
| 軽量なコンテキスト | 必要な情報を必要な時にのみ取得する                       |
| 自己進化           | スキルは使用されるたびにフィードバックを蓄積し改善される |

## 3層アーキテクチャ

### 層の定義

| 層            | ディレクトリ           | 責務                                   | AI/確定             | 詳細仕様                           |
| ------------- | ---------------------- | -------------------------------------- | ------------------- | ---------------------------------- |
| Interface層   | `.claude/commands/ai/` | ユーザー入力受付、エージェント起動指示 | AI                  | [18-commands.md](./18-commands.md) |
| Application層 | `.claude/agents/`      | ワークフロー制御、判断、スキル参照     | AI                  | [18-agents.md](./18-agents.md)     |
| Domain層      | `.claude/skills/`      | 知識提供、確実な実行、出力テンプレート | 読取専用 + 確定実行 | [18-skills.md](./18-skills.md)     |

### 依存関係ルール

**依存の方向**（以下の方向のみ許可、逆方向は禁止）:

| 依存元  | 依存先                            |
| ------- | --------------------------------- |
| Command | Agent（起動指示のみ）             |
| Agent   | Skill（知識参照、スクリプト実行） |
| Skill   | なし（外部依存ゼロ）              |

**禁止事項**:

- Command が Skill を直接参照すること
- Agent が他の Agent に依存すること（Task ツール経由の協調は許可）
- Skill が Agent や Command を参照すること
- 循環依存の形成

### 各層の責務境界

**Command層が行うこと/行わないこと**:

| 行うこと                   | 行わないこと           |
| -------------------------- | ---------------------- |
| ユーザー向け説明の提供     | ワークフローの詳細定義 |
| 引数の定義と受付           | ビジネスロジックの実装 |
| 起動するエージェントの指定 | スキルの直接参照       |
| 期待される成果物の宣言     | ファイル操作の実行     |

**Agent層が行うこと/行わないこと**:

| 行うこと                     | 行わないこと             |
| ---------------------------- | ------------------------ |
| ワークフロー（Phase）の制御  | 詳細なドメイン知識の保持 |
| スキルの参照と知識の取得     | 知識の永続的な保存       |
| スクリプトの実行指示         | ユーザー入力の直接受付   |
| 判断と意思決定               |                          |
| エラーハンドリング           |                          |
| 他エージェントへのハンドオフ |                          |

**Skill層が行うこと/行わないこと**:

| 行うこと                           | 行わないこと         |
| ---------------------------------- | -------------------- |
| ドメイン知識の提供                 | ワークフローの制御   |
| 判断基準・ルールの定義             | 他スキルの実行       |
| スクリプトによる確実な実行         | エージェントへの指示 |
| テンプレートによる出力形式の標準化 |                      |
| 変更履歴の記録                     |                      |
| 評価基準の定義                     |                      |

## 各層の詳細仕様

### Command 層（Interface 層）

**詳細仕様**: [18-commands.md](./18-commands.md)

**主要トピック**:

- YAML Frontmatter 設計（description, allowed-tools）
- エージェント起動フローの設計
- ツール権限設定（最小権限の原則）
- command_list.md による発見性の向上

### Agent 層（Application 層）

**詳細仕様**: [18-agents.md](./18-agents.md)

**主要トピック**:

- YAML Frontmatter 設計（name, description, tools, model）
- 役割定義（モデル人物、参考文献）
- ワークフロー設計（Phase別の目的・背景・ゴール）
- スキル依存関係のPhase別マッピング

### Skill 層（Domain 層）

**詳細仕様**: [18-skills.md](./18-skills.md)

**主要トピック**:

- 標準フォルダ構造（SKILL.md, EVALS.json, CHANGELOG.md, LOGS.md, scripts/, resources/, templates/）
- 参照書籍による知識圧縮
- Progressive Disclosure（Level 1-4）
- フィードバックループ（log_usage.mjs）
- skill_list.md によるエージェント別スキル管理

## 共通仕様

### 命名規則

| 対象           | 規則                                                            |
| -------------- | --------------------------------------------------------------- |
| コマンド名     | 動詞ベース、ケバブケース（例: `create-agent`）                  |
| エージェント名 | 役割ベース、ケバブケース（例: `meta-agent-designer`）           |
| スキル名       | ドメイン+トピック、ケバブケース（例: `agent-structure-design`） |

### ファイル参照形式

**必須形式**: プロジェクトルートからの相対パス（`.claude/` で始まる形式）

| 参照対象     | パス形式                                              |
| ------------ | ----------------------------------------------------- |
| エージェント | `.claude/agents/{{agent-name}}.md`                    |
| スキル       | `.claude/skills/{{skill-name}}/SKILL.md`              |
| コマンド     | `.claude/commands/ai/{{command-name}}.md`             |
| リソース     | `.claude/skills/{{skill-name}}/resources/{{file}}.md` |
| スクリプト   | `.claude/skills/{{skill-name}}/scripts/{{file}}.mjs`  |

### バージョン管理

すべてのコンポーネントはセマンティックバージョニング（MAJOR.MINOR.PATCH）で管理される。

| バージョン要素 | 変更時                                 |
| -------------- | -------------------------------------- |
| MAJOR（X.0.0） | 後方互換性のない変更（構造変更、削除） |
| MINOR（0.X.0） | 後方互換性のある機能追加               |
| PATCH（0.0.X） | バグ修正、軽微な改善                   |

### エラーハンドリング

| 回復方法                    | 確実性 | 備考                       |
| --------------------------- | ------ | -------------------------- |
| Git操作によるロールバック   | 確実   | `git restore`、`git reset` |
| スクリプトの exit code 確認 | 確実   | 失敗検知が可能             |
| リトライ                    | 部分的 | 同じ結果になるとは限らない |
| AI による自動診断           | 不確実 | 誤診断の可能性あり         |

### 状態管理

Claude Code はセッションベースで動作するため、永続的な状態管理は行わない。

**状態の受け渡し方法**:

| 方法                | 使用場面                     |
| ------------------- | ---------------------------- |
| Task ツールの戻り値 | エージェント間の情報受け渡し |
| ファイル生成        | 成果物の永続化               |
| LOGS.md への追記    | 実行記録の保存               |

## 用語定義

| 用語                         | 定義                                                  |
| ---------------------------- | ----------------------------------------------------- |
| Command                      | ユーザーがスラッシュ（/）で起動するエントリーポイント |
| Agent                        | タスクを実行する専門家エージェント                    |
| Skill                        | エージェントが参照する知識とスクリプトの集合体        |
| Progressive Disclosure       | 必要に応じて段階的に情報を開示する設計手法            |
| フィードバックループ         | 実行結果を評価し改善につなげる仕組み                  |
| セマンティックバージョニング | MAJOR.MINOR.PATCH 形式のバージョン管理                |
| ハンドオフ                   | エージェント間の処理引き継ぎ                          |
| Phase                        | ワークフローの段階                                    |

## クイックリファレンス

### Command 作成時のチェックリスト

- [ ] YAML Frontmatter が有効（description, allowed-tools）
- [ ] description に起動エージェントが明記されている
- [ ] allowed-tools が最小権限
- [ ] 本文に目的・背景・ゴール・エージェント起動フローがある

**詳細**: [18-commands.md](./18-commands.md)

### Agent 作成時のチェックリスト

- [ ] YAML Frontmatter が有効（name, description, tools, model）
- [ ] description にモデル人物が明記されている
- [ ] description に依存スキルが列挙されている（📚 依存スキル (N個)）
- [ ] 本文に目的・責務・背景・役割定義・ワークフローがある
- [ ] 各Phaseに目的・背景・ゴール・読み込むスキル・完了条件がある
- [ ] 最終Phaseに「記録と評価」が含まれている
- [ ] 行数が450-550行範囲内

**詳細**: [18-agents.md](./18-agents.md)

### Skill 作成時のチェックリスト

- [ ] SKILL.md が500行以内
- [ ] YAML Frontmatter が有効（name, description, version, level, last_updated, references）
- [ ] EVALS.json が存在し、有効なJSON
- [ ] CHANGELOG.md が存在
- [ ] LOGS.md が存在
- [ ] scripts/log_usage.mjs が存在
- [ ] 本文に概要・ワークフロー・ベストプラクティス・変更履歴がある

**詳細**: [18-skills.md](./18-skills.md)
