## Skill構造仕様

### 標準フォルダ構造（改訂：agents/ を追加）

```
skill-name/
├── SKILL.md              # 必須: 中枢（500行以内推奨）
├── agents/               # 推奨: Task仕様書（実行直前にロードされる前提で軽量に）
│   ├── task-1.md
│   └── task-n.md
├── scripts/              # 任意: 実行可能コード（Python/Bash/Node等）
│   └── log_usage.mjs     # 推奨: フィードバックループ用標準スクリプト
├── references/           # 任意: 必要時に読む参照資料（知識の外部化）
└── assets/               # 任意: 出力で使用する素材（テンプレ/雛形等）
```

**重要**: 以下のファイルは作成しない（SkillはAI実行のための情報のみ）

- README.md
- INSTALLATION_GUIDE.md
- QUICK_REFERENCE.md
- その他の補助ドキュメント

## SKILL.md 仕様

### YAML Frontmatter

**公式フィールド（Anthropic準拠）**:

| フィールド      | 型       | 必須 | 説明                                       |
| --------------- | -------- | ---- | ------------------------------------------ |
| `name`          | string   | 必須 | スキル識別子（ハイフンケース、最大64文字） |
| `description`   | string   | 必須 | スキル説明と発動条件（最大1024文字）       |
| `license`       | string   | 任意 | ライセンス情報                             |
| `allowed-tools` | string[] | 任意 | 許可ツール                                 |
| `metadata`      | object   | 任意 | 追加メタ                                   |

**プロジェクト拡張フィールド（改訂）**:

| フィールド     | 型       | 必須 | 説明                                                |
| -------------- | -------- | ---- | --------------------------------------------------- |
| `tags`         | string[] | 任意 | 検索用タグ                                          |
| `dependencies` | string[] | 任意 | 依存スキルのパス（※参照のみ。自動実行を意味しない） |

> **削除**: `references:` フィールド（書籍等の構造化リスト）は本プロジェクトでは採用しない。
> 理由：description 内にアンカーとして統合し、トリガー性能（発見性）を優先するため。

### name フィールドの規則

| 規則                     | 説明                         |
| ------------------------ | ---------------------------- |
| ハイフンケース           | 小文字、数字、ハイフンのみ   |
| 最大64文字               | 長すぎる名前は不可           |
| 先頭・末尾にハイフン不可 | `my-skill` ○、`-my-skill-` × |
| 連続ハイフン不可         | `my-skill` ○、`my--skill` ×  |
| ディレクトリ名と一致     | フォルダ名と完全一致         |

### description フィールド（最重要：トリガー設計）

**descriptionはスキルの主要なトリガーメカニズム。**
Skillは段階的ロードされるため、発見フェーズでは本文より description が効く。

| 規則                          | 説明                                                                 |
| ----------------------------- | -------------------------------------------------------------------- |
| 最大1024文字                  | 簡潔かつ包括的に                                                     |
| 角括弧禁止                    | `<` `>` は使用不可                                                   |
| 「何を」と「いつ」を含める    | 機能＋発動条件を必ず書く                                             |
| 本文に When to Use 不要       | 発動条件は description に集約                                        |
| アンカーは description に統合 | ドキュメント/リファレンス/スイーズ情報は **Markdown禁止で** 埋め込む |

**description 記述形式（Markdown禁止版）**
※ `-` や `*` による箇条書きは使わない。行区切りで表現する。

```yaml
description: |
  {{スキルの概要（2〜3行）}}

  Anchors:
  • {{アンカー名}} / 適用: {{適用範囲}} / 目的: {{目的}}
  • {{アンカー名}} / 適用: {{適用範囲}} / 目的: {{目的}}

  Trigger:
  Use when {{英語での発動条件（短く、具体例語彙を含める）}}.
```

> Anchors には「書籍」「社内ルール」「スキーマ」「Runbook」「API仕様」「設計方針」などを混在させてよい。
> ただし **"適用範囲" と "目的" を必ず添える**。

### 本文（Markdown Body）

Frontmatter の後に続く指示とガイダンス。スキル発動後にのみ読まれる。

**推奨セクション構造**:

| セクション              | 必須 | 説明                                         |
| ----------------------- | ---- | -------------------------------------------- |
| 概要                    | 必須 | 1-2文で目的                                  |
| ワークフロー            | 推奨 | Phase ベースの流れ（Task分割の案内を含める） |
| Task仕様（ナビ）        | 必須 | agents/ の各Taskをいつ呼ぶか、入出力は何か   |
| ベストプラクティス      | 必須 | すべきこと/避けるべきこと                    |
| リソース/スクリプト参照 | 推奨 | references/ scripts/ assets/ への導線        |

**行数制約**: 500行以内。超過時は references/ に分離。
