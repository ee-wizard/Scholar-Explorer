## 記述ガイドライン

> このセクションは実際の仕様作成時に削除してください

### するべきこと

- 文章による説明（設計意図・目的を明確に）
- 表形式でデータ構造・設定項目を整理
- 箇条書きで手順・要件をリスト化
- 誰でも理解できる粒度で記述
- 見出しは命名ベースで管理（番号なし）

### 避けるべきこと

- ソースコードの直接記述（TypeScript, JSON等）
- 実装詳細への偏り
- 特定実装に依存した説明
- 専門用語の説明なしでの使用
- 見出しへの番号付け（例: `## 1. 概要` → `## 概要`）

### ファイル命名規則

- 形式: `{prefix}-{topic}.md`
- 例: `architecture-database.md`, `api-endpoints.md`
- 番号付けは不可
