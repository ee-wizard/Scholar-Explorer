Understanding the four types of documentation is crucial for writing helpful content.

## 1. Tutorials (The Teacher)
**Goal**: Acquiring competence.
**Structure**:
- "In this tutorial, you will build..."
- Step 1: Install...
- Step 2: Run...
- Step 3: See result...
**Do**: Focus on success. Give immediate results.
**Don't**: Explain abstract concepts. Show edge cases.

## 2. How-to Guides (The Cookbook)
**Goal**: Solving a problem.
**Structure**:
- "How to configure CORS"
- "Prerequisites"
- "Steps"
**Do**: Be specific. Focus on the result.
**Don't**: Teach basic concepts (assume competence).

## 3. Reference (The Dictionary)
**Goal**: Information retrieval.
**Structure**:
- Function name
- Parameters
- Return value
- Exceptions
**Do**: Be accurate. Be consistent.
**Don't**: Tell a story.

## 4. Explanation (The Professor)
**Goal**: Understanding.
**Structure**:
- "Background"
- "Architecture"
- "Trade-offs"
**Do**: Give context. Explain "Why".
**Don't**: Give instructions.

---

**Before writing, ask yourself**: "Is the user trying to learn (Tutorial), do (How-to), look up (Reference), or understand (Explanation)?"
