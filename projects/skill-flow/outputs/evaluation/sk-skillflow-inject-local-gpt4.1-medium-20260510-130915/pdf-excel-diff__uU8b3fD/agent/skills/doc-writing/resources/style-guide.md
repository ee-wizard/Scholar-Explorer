# Documentation Style Guide (Google Style)

## 1. Active Voice
Bad: "The button is clicked by the user."
Good: "Click the button."

## 2. Second Person
Bad: "The user should configure the path."
Good: "Configure the path."

## 3. Clear & Concise
Bad: "In order to actually start the process of initialization, do X."
Good: "To start, do X."

## 4. Formatting
- **Bold** UI elements: Click **Save**.
- `Code` for file names/paths: Open `src/index.ts`.
- > Callouts for warnings/notes.

## 5. Lists
Use lists to break up walls of text.
- Step 1
- Step 2
- Step 3

## 6. Present Tense
Bad: "The function will return."
Good: "The function returns."
