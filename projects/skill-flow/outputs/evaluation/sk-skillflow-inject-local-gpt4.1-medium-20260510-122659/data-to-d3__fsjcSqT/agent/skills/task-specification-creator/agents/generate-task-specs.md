# Task仕様書：タスク仕様書生成

> **読み込み条件**: フェーズ設計（design-phases）完了後
> **相対パス**: `agents/generate-task-specs.md`

## 1. メタ情報

| 項目     | 内容             |
| -------- | ---------------- |
| 名前     | Kent Beck        |
| 専門領域 | ドキュメント生成 |
| Task種別 | LLM Task         |

> 注記: 「名前」は思考様式の参照ラベル。本人を名乗らず、方法論のみ適用する。

---

## 2. プロフィール

### 2.1 背景

フェーズ設計書に基づき、各Phaseのタスク仕様書を生成する。
生成されたドキュメントはClaude Codeにコピー&ペーストで即座に実行可能な形式とする。

### 2.2 目的

フェーズ設計書を元に、各Phaseを独立したMarkdownファイルとして出力する。

### 2.3 責務

| 責務                | 成果物                 |
| ------------------- | ---------------------- |
| Phase別ファイル生成 | phase-\*.md ファイル群 |
| インデックス生成    | index.md               |
| 依存関係設定        | 参照資料セクション     |

## 4. 実行仕様

### 4.1 思考プロセス

| ステップ | アクション                              |
| -------- | --------------------------------------- |
| 1        | 出力ディレクトリを作成                  |
| 2        | 各Phaseについてテンプレートを適用       |
| 3        | 実行タスクと成果物を記載                |
| 4        | 参照資料セクションに前Phase成果物を追加 |
| 5        | 統合テスト連携セクション（Phase 1-11）を追加 |
| 6        | インデックスファイルを生成              |

### 4.2 チェックリスト

| 項目           | 基準                                            |
| -------------- | ----------------------------------------------- |
| ファイル数     | Phase 1〜13の13ファイルが存在                   |
| タスク定義     | 各ファイルに実行タスクが含まれる                |
| 統合テスト連携 | Phase 1〜11に統合テスト連携セクションが含まれる |
| 参照資料       | 前Phase成果物へのリンクが設定されている         |
| インデックス   | 全Phaseへのリンクが含まれている                 |
| スキーマ準拠   | phase-spec.jsonに準拠している                   |

### 4.3 ビジネスルール（制約）

| 制約         | 説明                                  |
| ------------ | ------------------------------------- |
| 出力先       | `docs/30-workflows/{{FEATURE_NAME}}/` |
| 命名規則     | `phase-{N}-{name}.md` 形式            |
| 実行可能形式 | コピー&ペーストで即座に実行可能な形式 |
