**責務**: 相手分析・目的設定・BSEC構成・ボリューム設計

---

## 1. 人を動かす資料の原則

### 3つの基本要素

| 要素 | 内容 | 質問 |
|------|------|------|
| 一人歩きする資料 | 説明者がいなくても伝わる | 資料だけで理解できるか？ |
| 相手分析 | 期待する行動を決める | 誰に何をしてほしいか？ |
| 自己分析 | 伝えることを決める | 相手は自分をどう見ているか？ |

## 4. BSEC構成（基本構成）

### 4つのセクション

```
B: Background（背景）
│  → 資料の重要性を示す
│  → なぜ今この話をするのか
│
S: Situation/Problem（課題）
│  → 問題点を示す
│  → 相手が共感できる課題提示
│
E: Explanation/Solution（解決策）
│  → 課題に対応する解決策
│  → 具体的なアクション
│
C: Conclusion/Effect（効果・結論）
   → 解決策の結果を示す
   → 期待する行動を明示
```

### 各セクションの役割

| セクション | 役割 | 答える質問 |
|-----------|------|-----------|
| 背景 | 必要性を示す | なぜ今この話？ |
| 課題 | 問題を共有 | 何が問題？ |
| 解決策 | 解決方法を提示 | どう解決する？ |
| 効果・結論 | 結果と行動を示す | どうなる？何をする？ |

### BSEC設計ワークシート

## 5. 資料の要となる4要素

| 要素 | 位置 | 役割 | 必須 |
|------|------|------|------|
| タイトル | 表紙 | 資料全体の主題を明示 | ◎ |
| サマリー | 冒頭（タイトル直後） | 結論と要点を先出し | ◎ |
| 目次 | サマリー直後 | 全体構成を俯瞰 | ○ |
| 結論 | 最終スライド | 期待する行動を明示 | ◎ |

## 6. ボリューム設計

### 相手タイプ別ページ数

| 相手タイプ | 推奨ページ数 | 詳細度 | 重点 |
|-----------|-------------|--------|------|
| 経営層・決裁者 | 5-8ページ | 概要中心 | 結論・数字・効果 |
| 実務担当者 | 10-15ページ | 詳細含む | 手順・スケジュール |
| 専門家 | 8-12ページ | 根拠重視 | データ・論理 |
| 初心者・一般 | 8-10ページ | 平易に | 具体例・図解 |

### 8ページ標準構成

```
1. 表紙（タイトル）
2. サマリー（結論先出し）
3. 目次
4. 背景（1-2枚）
5. 課題（1-2枚）
6. 解決策（1-2枚）
7. 効果（1枚）
8. 結論（期待する行動）
```

## 7. セクション構成

### セクションごとの目次スライド

各セクションの冒頭に目次スライドを配置し、現在位置を示す。

```
┌─ 表紙
├─ サマリー
├─ 目次
│
├─ セクション1: 背景
│   ├─ 【目次スライド】★現在位置を強調
│   └─ 背景スライド
│
├─ セクション2: 課題
│   ├─ 【目次スライド】★現在位置を強調
│   └─ 課題スライド
│
├─ セクション3: 解決策
│   ├─ 【目次スライド】★現在位置を強調
│   └─ 解決策スライド
│
├─ セクション4: 効果
│   ├─ 【目次スライド】★現在位置を強調
│   └─ 効果スライド
│
└─ 結論
```

### パンくずリストによる流れ表示

スライド上部またはフッターに現在位置を表示：

```
背景 > 課題 > 【解決策】 > 効果 > 結論
```
