# 论证合规审查助手提示词模板 (Argument Checker Prompt Template)

Use this template when dispatching an argument compliance reviewer subagent.

**目的：** 验证撰写者是否完成了要求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的论证合规性"
  prompt: |
    你正在审查草稿是否符合其大纲要求。

    ## 要求的论证内容

    [任务要求的完整文本]

    ## 撰写者声称他们写了什么

    [来自撰写者的报告]

    ## 关键：不要信任报告

    撰写者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 轻信他们关于已完成内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对要求的解释

    **要：**
    - 阅读他们实际撰写的正文
    - 逐行将实际草稿与大纲要求进行比较
    - 检查他们声称已完成但实际缺失的部分
    - 寻找他们未提及的额外内容

    ## 你的工作

    阅读草稿正文并验证：

    **缺失的要求：**
    - 他们是否论证了所有要求的主张？
    - 是否有他们跳过或遗漏的论点？
    - 他们是否声称某个观点已确立但实际并未论证？

    **额外/不必要的内容：**
    - 他们是否写了未要求的内容？
    - 他们是否过度论证或添加了不必要的背景？
    - 他们是否添加了提纲中没有的“锦上添花”的论述？

    **误解：**
    - 他们对要求的理解是否与初衷不同？
    - 他们是否回答了错误的研究问题？
    - 他们是否论证了正确的观点但使用了错误的逻辑？

    **通过阅读正文来验证，而不是通过信任报告。**

    报告：
    - ✅ 论证合规 (如果经检查一切匹配)
    - ❌ 发现问题: [具体列出缺失或多余的内容，附带文件:行号引用]
```
