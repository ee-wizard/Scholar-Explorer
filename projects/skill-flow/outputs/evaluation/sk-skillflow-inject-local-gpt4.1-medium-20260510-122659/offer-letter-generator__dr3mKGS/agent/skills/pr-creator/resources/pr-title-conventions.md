# PR Title Conventions

PR titles follow the same Conventional Commits format as commit messages for consistency.

## Format

```
<type>[(scope)][!]: <description>
```

## Components

### Type (Required)

| Type | Description | Example |
|------|-------------|---------|
| `feat` | New feature | `feat: add user auth` |
| `fix` | Bug fix | `fix: resolve null pointer` |
| `docs` | Documentation | `docs: update README` |
| `style` | Formatting only | `style: apply ktfmt` |
| `refactor` | Code restructure | `refactor: extract service` |
| `perf` | Performance | `perf: optimize query` |
| `test` | Tests | `test: add unit tests` |
| `build` | Build system | `build: upgrade Gradle` |
| `ci` | CI/CD | `ci: add matrix build` |
| `chore` | Maintenance | `chore: update deps` |
| `revert` | Revert change | `revert: feat(api): change` |

### Scope (Optional)

Indicates the area of the codebase affected.

**Fakt-specific scopes:**
| Scope | When to Use |
|-------|-------------|
| `compiler` | Core compiler plugin |
| `fir` | FIR phase changes |
| `ir` | IR phase changes |
| `generation` | Code generation |
| `analysis` | Interface analysis |
| `dsl` | Configuration DSL |
| `types` | Type resolution |
| `gradle` | Gradle plugin |
| `annotations` | @Fake annotation |
| `samples` | Sample projects |
| `docs` | Documentation |

### Breaking Change Indicator (Optional)

Add `!` before `:` to indicate breaking changes:

```
feat(api)!: change factory signature
refactor(dsl)!: restructure configuration
```

### Description

- **Imperative mood**: "add" not "added"
- **No period** at end
- **Lowercase** after colon (unless proper noun)
- **Max 72 characters** total

## Examples

### Good Titles

```
feat(compiler): add generic type parameter support
fix(fir): resolve annotation detection for nested interfaces
docs: update KMP multi-module setup guide
refactor(generation): extract factory to separate class
perf(analysis): cache interface metadata
test(compiler): add GIVEN-WHEN-THEN tests for generics
build: upgrade Kotlin to 2.1.0
ci: add KMP platform matrix
chore(deps): update kotlinx-coroutines
feat(dsl)!: require explicit configuration block
```

### Bad Titles (Avoid)

```
# Too vague
Update stuff
Fix bug
Changes

# Wrong mood
feat: added new feature
fix: fixed the issue

# Too long
feat(compiler): implement comprehensive generic type parameter support for all interface method variations including bounds and variance

# With period
feat: add feature.

# Missing type
add generic support

# Invalid type
feature: add thing
bugfix: fix thing

# AI attribution (FORBIDDEN)
feat: add feature [Generated by Claude]
```

## Multi-Commit PRs

When a PR contains multiple commits of different types:
- Use the **most significant** type
- Or the type that best represents the **primary purpose**

Example: PR with `feat` + `test` + `docs` commits:
```
feat(compiler): add suspend function support
```

## Breaking Changes

Two ways to indicate:

**1. With `!` suffix (preferred for titles):**
```
feat(api)!: change factory return type
```

**2. Mentioned in description:**
```
refactor(dsl): restructure configuration API

BREAKING CHANGE: Configuration now requires explicit block.
```

## Character Count Guide

```
         1         2         3         4         5         6         7
123456789012345678901234567890123456789012345678901234567890123456789012
                                                  |50 char   |72 char
```

- **Aim for**: 50 characters
- **Maximum**: 72 characters

## Quick Reference

```
feat(scope): add new functionality
fix(scope): resolve specific bug
docs: update documentation only
style: formatting changes only
refactor(scope): restructure code, same behavior
perf(scope): improve performance
test(scope): add or fix tests
build: build system changes
ci: CI/CD configuration
chore: maintenance tasks
revert: undo previous change
```
