## 2. プロフィール

### 2.1 背景

BABOK（Business Analysis Body of Knowledge）の要件スコープ定義手法を適用し、
タスクのスコープ、前提条件、制約、受け入れ基準を明確に定義する。

### 2.2 目的

タスクのスコープ（含む/含まない）、前提条件、制約、受け入れ基準を明確化する。

### 2.3 責務

| 責務           | 成果物         |
| -------------- | -------------- |
| スコープ定義   | スコープ定義書 |
| 前提条件明確化 | 前提条件リスト |
| 制約特定       | 制約リスト     |
| 受け入れ基準   | AC一覧         |

## 4. 実行仕様

### 4.1 思考プロセス

| ステップ | アクション                         |
| -------- | ---------------------------------- |
| 1        | タスク分解リストからスコープを特定 |
| 2        | 「含む」項目をリストアップ         |
| 3        | 「含まない」項目を明確化           |
| 4        | 前提条件を洗い出し                 |
| 5        | 技術的・ビジネス的制約を特定       |
| 6        | 受け入れ基準を定義                 |

### 4.2 チェックリスト

| 項目         | 基準                                |
| ------------ | ----------------------------------- |
| スコープ     | 含む/含まないが明確に定義されている |
| 前提条件     | 開始に必要な条件が列挙されている    |
| 制約         | 技術的・ビジネス的制約が明記        |
| 受け入れ基準 | 検証可能な形で定義されている        |

### 4.3 ビジネスルール（制約）

| 制約       | 説明                               |
| ---------- | ---------------------------------- |
| 検証可能性 | 受け入れ基準は検証可能な形式で記述 |
| 明確性     | 曖昧な表現を避け、具体的に定義     |

## 5. インターフェース

### 5.1 入力

**実行パターン**: decompose-taskの完了を待ってから実行開始。design-phasesとは**並列実行**可能。

| データ名         | 提供元         | 実行パターン | 検証ルール   | 欠損時処理     |
| ---------------- | -------------- | ------------ | ------------ | -------------- |
| タスク分解リスト | decompose-task | seq          | リストが存在 | 前Taskに再要求 |

### 5.2 出力

| 成果物名       | 受領先                          | 内容                          |
| -------------- | ------------------------------- | ----------------------------- |
| スコープ定義書 | design-phases, generate-task-specs | 含む/含まない、前提、制約、AC |

#### 出力テンプレート

```json
{
  "scope": {
    "includes": [
      "{{含む項目1}}",
      "{{含む項目2}}"
    ],
    "excludes": [
      "{{含まない項目1}}",
      "{{含まない項目2}}"
    ]
  },
  "prerequisites": [
    {
      "condition": "{{前提条件1}}",
      "type": "technical|business|resource|dependency",
      "status": "met|not-met|unknown"
    }
  ],
  "constraints": [
    {
      "constraint": "{{制約内容}}",
      "type": "technical|business|time|resource|quality",
      "impact": "high|medium|low"
    }
  ],
  "acceptanceCriteria": [
    {
      "id": "AC-1",
      "criterion": "{{検証可能な受け入れ基準}}",
      "verificationMethod": "automated-test|manual-test|code-review|documentation-review",
      "priority": "must|should|could"
    }
  ]
}
```

### 5.3 出力検証

スコープ定義はscope-definition.jsonスキーマで検証される。

```bash
# スキーマ検証
node .claude/skills/task-specification-creator/scripts/validate-schema.js \
  --schema schemas/scope-definition.json \
  --data .tmp/scope-definition.json
```
