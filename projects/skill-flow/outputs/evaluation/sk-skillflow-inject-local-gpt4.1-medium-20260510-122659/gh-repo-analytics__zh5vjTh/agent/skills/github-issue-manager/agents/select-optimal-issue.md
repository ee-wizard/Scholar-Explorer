# 最適Issue選択エージェント

> **Progressive Disclosure**
>
> - 読み込みタイミング: selectモード Phase 4
> - 読み込み条件: スコアリング結果を基にユーザーに推奨Issue提示時
> - 前提: `select_issue.js` によるスコアリング完了後

## 責務

スコアリング結果を基に、ユーザーに最適なIssueを推奨し、選択をサポートする。

## 入力

- `scoredIssues`: スコア付きIssue配列（上位5件）
- `context`: ユーザーの現在のコンテキスト（オプション）

## 実行手順

### Step 1: スコアリング結果の確認

```bash
node .claude/skills/github-issue-manager/scripts/select_issue.js
```

### Step 2: 上位Issue分析

上位5件のIssueについて以下を分析:

1. **スコア内訳**
   - 優先度: 高=100, 中=50, 低=25
   - 規模: 小=30, 中=20, 大=10
   - 依存満足: +50 / 未満足: -100
   - 経過日数: +1/日 (max 30)

2. **依存関係チェック**
   - 依存Issueが完了しているか
   - 依存未満足の場合、先に着手すべきIssueを提示

3. **コンテキスト適合性**（オプション）
   - 現在の作業領域との関連性
   - 技術スタック・ドメインの一致度

### Step 3: 推奨提示

#### 推奨フォーマット

```markdown
## 🎯 推奨Issue

### 第1候補: #{{NUMBER}} {{TITLE}}

**スコア**: {{SCORE}}点

**推奨理由**:

- {{REASON_1}}
- {{REASON_2}}
- {{REASON_3}}

**概要**:
{{SUMMARY}}

**依存関係**: {{DEPENDENCIES_STATUS}}

---

### 他の候補

| 順位 | #       | タイトル | スコア | 備考      |
| ---- | ------- | -------- | ------ | --------- |
| 2    | #{{N2}} | {{T2}}   | {{S2}} | {{NOTE2}} |
| 3    | #{{N3}} | {{T3}}   | {{S3}} | {{NOTE3}} |
```

### Step 4: ユーザー選択

AskUserQuestionで選択を求める:

```
選択してください:
1. #{{NUMBER}} - {{TITLE}} (推奨)
2. #{{NUMBER2}} - {{TITLE2}}
3. #{{NUMBER3}} - {{TITLE3}}
4. 条件を変更して再検索
```

### Step 5: 選択後アクション

選択されたIssueに対して:

1. **ステータス更新**

   ```bash
   node .claude/skills/github-issue-manager/scripts/update_issue.js \
     --number {{NUMBER}} --status in-progress
   ```

2. **仕様書読み込み**
   - ローカルIssueファイルまたは元仕様書を読み込み
   - 実装開始のコンテキストを構築

## 出力

- 選択されたIssue番号
- Issue詳細（メタ情報・本文）
- 次のアクション提案

## 判断基準

### 最優先で選択すべきケース

1. **依存関係がすべて満足 + 高優先度**
   - 即座に着手可能で重要

2. **小規模 + 高優先度**
   - クイックウィンが期待できる

3. **経過日数が長い + 依存満足**
   - 長期放置を解消

### 選択を避けるべきケース

1. **依存関係未満足**
   - 先に依存Issueを完了すべき
   - ただし依存Issueを代わりに推奨

2. **大規模 + 低優先度**
   - 投資対効果が低い

## エラーハンドリング

| 状況           | 対応                   |
| -------------- | ---------------------- |
| 該当Issue 0件  | フィルタ条件緩和を提案 |
| 全て依存未満足 | 依存解消順序を提案     |
| 同スコアが複数 | 規模が小さい方を優先   |

## 使用例

```
User: 次に着手すべきIssueを選んで
```
