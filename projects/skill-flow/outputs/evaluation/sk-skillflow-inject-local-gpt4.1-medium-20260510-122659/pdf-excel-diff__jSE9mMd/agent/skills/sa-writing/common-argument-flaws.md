# 论证逻辑谬误 (Argumentation Fallacies)

**加载此参考当：** 撰写或修改论证、添加引用，或试图在正文中添加仅为了通过逻辑检查的内容时。

## 概述

验证必须针对真实的逻辑，而不是稻草人。引用是为了支持论点，而不是为了引用而引用。

**核心原则：** 检验你的论证逻辑是否真实有效，而不是看起来有效。

**遵循严格的 ADW (论证驱动写作) 可防止这些谬误。**

## 铁律

```
1. 绝不攻击稻草人 (验证虚假的对立观点)
2. 绝不在正文中添加仅为了通过检查的句子
3. 绝不在未阅读的情况下引用文献
```

## 谬误 1: 攻击稻草人 (Attacking a Strawman)

**违规：**
```markdown
<!-- ❌ 错误：验证一个明显错误的对立观点 -->
主张：地球不是平的。
证据：如果地球是平的，猫会把所有东西推下去。
```

**为什么这是错的：**
- 你在反驳一个容易被推翻的荒谬观点，而不是真正的学术对立观点
- 验证通过了，但这证明不了什么
- 没有增加学术价值

**修正：**
```markdown
<!-- ✅ 正确：验证真实的学术争议 -->
主张：尽管地平说有历史背景，但现代大地测量学提供了确凿的球体证据。
证据：卫星图像、重力测量数据...
```

## 谬误 2: 为了论证而论证 (Circular Reasoning)

**违规：**
```markdown
<!-- ❌ 错误：在正文中定义结论 -->
根据定义，方案 A 是最好的，因为我们将方案 A 定义为最佳方案。
```

**为什么这是错的：**
- 正文被自我引用的逻辑污染
- 看起来逻辑严密，实则空洞
- 违反了学术严谨性

**修正：**
```markdown
<!-- ✅ 正确：通过外部证据支持结论 -->
方案 A 在三个关键指标上优于方案 B... (引用实验数据)
```

## 谬误 3: 引用未读文献 (Citing Unread Sources)

**违规：**
```markdown
<!-- ❌ 错误：为了显得有依据而堆砌引用 -->
许多研究支持这一观点 (Smith 2010; Jones 2012; Wang 2015)。
<!-- 实际上这些文献可能根本没说这个 -->
```

**为什么这是错的：**
- 引用可能与你的论点无关甚至相悖
- 虚假的学术繁荣
- 读者一查原文就会发现破绽

**修正：**
```markdown
<!-- ✅ 正确：准确引用 -->
Smith (2010) 指出... 但 Jones (2012) 提出了反驳...
```

### 门控功能

```
在添加任何引用之前：
  停止 - 先别引用

  1. 问：“我读过这篇文献吗？”
  2. 问：“它真的支持我的论点吗？”
  3. 问：“我是否断章取义？”

  如果未读过：
    停止 - 阅读摘要和结论
    或者不要引用

  如果只是为了凑数：
    停止 - 删除引用
```

## 谬误 4: 不完整的论证 (Incomplete Argumentation)

**违规：**
```markdown
<!-- ❌ 错误：只展示有利证据 -->
数据表明收入增加了。
<!-- 忽略了通货膨胀更高的数据 -->
```

**为什么这是错的：**
- **片面真相** - 隐藏了完整图景
- **读者会发现漏洞** - 损害信誉
- **虚假信心** - 看起来论证有力，实则脆弱

**铁律：** 展示完整的证据链，包括可能削弱你观点的反例（并解释之）。

**修正：**
```markdown
<!-- ✅ 正确：展示完整图景 -->
虽然名义收入增加了 5%，但考虑到 6% 的通胀率，实际购买力略有下降。这表明...
```

## 谬误 5: 验证作为事后诸葛亮 (Verification as Afterthought)

**违规：**
```
✅ 正文撰写完成
❌ 没有进行逻辑检查
“准备好提交了”
```

**为什么这是错的：**
- 验证是撰写的一部分，不是可选的后续步骤
- ADW 本可以发现逻辑漏洞
- 不能在未验证逻辑的情况下声称完成

**修正：**
```
ADW 循环：
1. 提出待证主张
2. 撰写证据以支持
3. 完善行文
4. 然后才声称完成
```

## ADW 防止这些谬误

**为什么 ADW 有帮助：**
1. **先提出主张** → 迫使你思考你要论证什么
2. **看它缺乏支持** → 确认你需要真实的证据，而不是稻草人
3. **最简论证** → 防止为了凑字数而废话
4. **真实证据** → 你在引用之前会看到需要什么证据

**如果你在验证虚假的逻辑，你就违反了 ADW** - 你在没有先确认主张缺乏真正支持的情况下就添加了论据。

## 快速参考

| 谬误 | 修正 |
|--------------|-----|
| 攻击稻草人 | 反驳最有力的对立观点 |
| 循环论证 | 使用外部证据支持 |
| 引用未读文献 | 阅读源文，准确引用 |
| 不完整论证 | 展示完整证据，包括反例 |
| 事后验证 | ADW - 先主张后撰写 |

## 底线

**引用和逻辑是用来支持真理的，而不是用来装饰门面的。**

如果 ADW 揭示你在验证虚假的逻辑，你就走偏了。

修正：论证真实的观点，或者质疑你为什么要写这一段。
