# ライティングルール

**責務**: タイトル・メッセージ・箇条書き・文章表現

---

## 1. スライドタイトルとメッセージ

### 役割分担

| 要素 | 役割 | 文字数 | ルール |
|------|------|--------|--------|
| タイトル | 内容を示す | 20文字以内 | 主張なし、簡潔に |
| メッセージ | 主張を伝える | 50文字以内 | 接続詞で流れを意識 |

### タイトルのルール

```
【NG】主張が入っている
× AI導入で30%のコスト削減が可能

【OK】内容のみを示す
○ AI導入の効果
○ コスト削減の試算
○ 市場環境の変化
```

### メッセージのルール

```
【NG】主張がない・長すぎる
× 市場環境について説明します
× AI導入によるコスト削減効果として人件費30%、運用コスト20%、保守コスト15%の削減が見込めます

【OK】50文字以内で主張
○ しかし、競合の参入により市場シェアが20%低下している
○ そのため、AI導入で競争力を回復させる必要がある
```

---

## 2. 接続詞による流れ作成

### 接続詞の使い分け

| 流れのタイプ | 接続詞 | 使用場面 |
|-------------|--------|---------|
| 順接 | そのため、したがって、よって | 論理展開・結論導出 |
| 逆接 | しかし、ただし、一方で | 課題提示・対比 |
| 追加 | さらに、加えて、また | 情報追加・補強 |
| 具体化 | たとえば、具体的には | 例示・詳細説明 |
| 結論 | このように、結果として | まとめ・締め |

### 流れの作成例

```
スライド1【背景】
タイトル: 市場環境の変化
メッセージ: デジタル化の加速により、業界全体で変革が求められている

スライド2【課題】
タイトル: 当社の現状
メッセージ: しかし、当社のデジタル化は遅れており、競争力が低下している

スライド3【解決策】
タイトル: 提案するソリューション
メッセージ: そのため、AIを活用した業務効率化を提案する

スライド4【効果】
タイトル: 導入効果
メッセージ: 結果として、コスト30%削減と生産性50%向上が見込める
```

## 3. 箇条書きのルール

### 作成手順

```
1. 分解から始める
   → 伝えたい内容を要素に分解

2. 絞り込む
   → 3項目に整理（多くても5項目）

3. 並び替える
   → 重要度順 / 時系列順 / 種類別

4. 文章化する
   → 一文40文字以内
   → 文末は用言か体言止め
```

### 3項目への整理

```
【NG】項目が多すぎる
・コスト削減
・生産性向上
・品質改善
・リスク低減
・スピード向上
・顧客満足度向上

【OK】3項目に整理
・コスト削減（30%）
・生産性向上（50%）
・品質改善（エラー率80%減）
```

### 並び順のパターン

| パターン | 使用場面 | 例 |
|---------|---------|-----|
| 重要度順 | 優先度を示す | 最重要 → 重要 → 参考 |
| 時系列順 | プロセスを示す | 準備 → 実行 → 検証 |
| 種類別 | カテゴリ分け | 戦略 / 組織 / システム |

## 4. 階層構造

### 階層のパターン

| 階層タイプ | 上位 | 下位 | 例 |
|-----------|------|------|-----|
| 因果 | 結果 | 原因 | 売上向上 ← 顧客増加 |
| 詳細 | 概要 | 詳細 | コスト削減 ← 人件費・運用費・保守費 |
| 事例 | 主張 | 具体例 | 効果あり ← A社・B社の事例 |

### 階層構造の表現

```
■ メイン項目（結論・主張）
  └ サブ項目1（因果：なぜそうなるか）
  └ サブ項目2（詳細：具体的には何か）
  └ サブ項目3（事例：どんな例があるか）

【例】
■ AI導入でコスト30%削減が可能
  └ 自動化により人件費が削減される
  └ 運用費・保守費も同時に削減
  └ A社では40%削減を達成
```

## 5. 文章表現ルール

### 文字数制限

| 要素 | 文字数 |
|------|--------|
| タイトル | 20文字以内 |
| メッセージ | 50文字以内 |
| 箇条書き1項目 | 40文字以内 |
| 説明文1文 | 60文字以内 |

### 文末の統一

| 表現タイプ | 文末 | 例 |
|-----------|------|-----|
| 用言止め | 動詞・形容詞 | 〜する、〜である |
| 体言止め | 名詞 | 〜の実現、〜の向上 |

**重要**: 同一スライド内で混在させない

```
【NG】混在
・コスト削減を実現する
・生産性の向上
・品質が改善される

【OK】体言止めで統一
・コスト削減の実現
・生産性の向上
・品質の改善

【OK】用言止めで統一
・コストを30%削減する
・生産性を50%向上させる
・品質を大幅に改善する
```

## 6. フォントサイズ（CSS変数）

### CSS変数一覧

| 用途 | CSS変数 | 説明 |
|------|---------|------|
| タイトル | `var(--fs-title)` | メインタイトル |
| サブタイトル | `var(--fs-subtitle)` | 副題 |
| 見出し | `var(--fs-heading)` | スライドタイトル |
| 小見出し | `var(--fs-subheading)` | セクション内見出し |
| 本文 | `var(--fs-body)` | 通常テキスト |
| 大きめ本文 | `var(--fs-body-lg)` | 強調テキスト |
| 小さめ文字 | `var(--fs-small)` | 補足・注釈 |

### ユーティリティクラス

| クラス | 用途 |
|--------|------|
| `.text-note` | 注釈・補足テキスト（グレー） |
| `.text-emphasis` | 強調テキスト |
| `.highlight` | ハイライト（黄色） |

## 7. 改行位置

### 原則

- 1行が長くなりすぎないよう、意味の切れ目で`<br>`を挿入
- 20文字を超える文章は改行を検討

### 改行すべき箇所

| 箇所 | 例 |
|------|-----|
| 20文字超の意味的区切り | テンプレートを使えば、<br>高品質な文書が作れる |
| 読点「、」の後 | コスト削減と、<br>生産性向上を実現 |
| 括弧の前後 | AI導入により<br>（3ヶ月で）効果を発揮 |
| 強調タグの前後 | これが<br><strong>最重要</strong>です |
